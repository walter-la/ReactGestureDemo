## 🎯 專案目標

這是一個 **React 單元件範例**，用來示範如何在手機（或支援觸控的裝置）上實作：

* 觸控拖曳（Drag）
* 滑動手勢偵測（Swipe，上下左右）
* 速度與距離門檻判斷
* UI 實際回饋與可互動控制

---

## 🧩 功能列表

### 1. 拖曳功能

* 使用者可以按住畫面中央的「拖曳我」卡片，並在互動區塊內隨手指移動。
* 拖曳過程會顯示即時位移資訊（dx/dy）、速度（v）、時間差（dt）。
* 是否「放開後回到中心」可透過設定開關切換：

  * **開啟** → 放開手指後卡片自動彈回原點。
  * **關閉** → 放開手指後卡片會停留在釋放位置。

### 2. 滑動手勢偵測

* 支援 **上下左右**滑動偵測。
* 判斷依據：

  * 滑動距離是否超過距離門檻（px）。
  * 滑動速度是否超過速度門檻（px/ms）。
* 偵測到滑動後，會在狀態列顯示滑動方向與速度。

### 3. 門檻調整

* 使用者可透過畫面上方的滑桿調整：

  * **距離門檻**（8 ~ 160px）
  * **速度門檻**（0 ~ 2 px/ms）
* 方便觀察不同門檻設定下的手勢偵測效果。

### 4. 方向按鈕

* 互動區底部有 **LEFT / RIGHT / UP / DOWN** 四個方向鍵。
* 點擊方向鍵可直接將卡片移動固定距離（例如 120px）。
* 最近一次觸發的方向會高亮顯示，方便測試。

### 5. 除錯與狀態顯示

* 中央資訊面板即時顯示：

  * 狀態（等待觸控、拖曳中、偵測到滑動…）
  * dx/dy 位移、速度、時間差。
* 方便學習與調試手勢判斷邏輯。

---

## ⚙️ 技術重點

* 採用 **Pointer Events API** → 一次支援觸控 / 滑鼠 / 手寫筆。
* 使用 `touch-action: none` → 避免瀏覽器預設捲動、手勢干擾。
* 透過 `setPointerCapture` → 確保快速移動時事件不中斷。
* 分離 **累積位移（basePos）** 與 **本次拖曳位移（dragPos）** → 邏輯清晰，保證可控。
